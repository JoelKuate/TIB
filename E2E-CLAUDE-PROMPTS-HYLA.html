🎭 Avec Playwright (code)

// test-purchase.spec.ts
test('Add product to cart', async ({ page }) => {
  await page.goto('https://hyla-air-solution.be');
  await page.click('text=Nos produits');
  await page.waitForSelector('.product-grid');
  await page.click('.product-card:first-child');
  await page.click('button[data-add-to-cart]');
  await expect(page.locator('.cart-count'))
    .toHaveText('1');
});

Scénario 1 — Navigation & Découverte produits

SC-01

Vérifie que l'utilisateur peut accéder au site, naviguer dans le catalogue et trouver un produit d'air purifié.

01
Navigation Accès à la page d'accueil
Prompt Claude Computer Use

Ouvre le navigateur et accède à l'URL https://hyla-air-solution.be/. Attends que la page soit complètement chargée (logo HYLA visible, menu de navigation présent). Si une bannière cookies apparaît, clique sur "Fermer" ou "Accepter" pour la fermer.

✅ Résultat attendu : La page d'accueil HYLA est visible avec le logo grenouille, le menu navigation et le contenu hero section.
Équivalent Playwright
await page.goto('https://hyla-air-solution.be/');
await page.waitForLoadState('networkidle');
await page.locator('.cookie-banner button').click();
02
Action Accès au catalogue produits
Prompt Claude Computer Use

Dans le menu de navigation principal, recherche et clique sur l'entrée correspondant aux produits (peut s'appeler "Nos produits", "Boutique", "Shop" ou un lien catalogue). Attends que la page liste de produits se charge.

✅ Résultat attendu : Une liste ou grille de produits (purificateurs d'air, aspirateurs, etc.) est affichée avec images et prix.
Équivalent Playwright
await page.click('nav >> text=/produits|boutique|shop/i');
await page.waitForSelector('[class*="product"]');
03
Assertion Vérification affichage catalogue
Prompt Claude Computer Use

VÉRIFIE que la page catalogue contient : — Au moins 1 produit visible avec une image — Un titre de produit lisible — Un prix affiché en euros (€) — Un bouton ou lien permettant d'accéder au détail du produit Prends une capture d'écran et décris ce que tu vois.

✅ Résultat attendu : N produits listés avec image, nom, prix €. Aucune erreur 404 ou message "aucun produit".
Équivalent Playwright
await expect(page.locator('.product-card')).toHaveCountGreaterThan(0);
await expect(page.locator('.price')).toContainText('€');

Scénario 2 — Fiche produit & Ajout au panier

SC-02

Vérifie qu'un utilisateur peut consulter le détail d'un produit et l'ajouter au panier.

04
Action Ouverture de la fiche produit
Prompt Claude Computer Use

Sur la page catalogue, clique sur le premier produit visible (clique sur l'image ou le titre du produit, pas sur un bouton "Ajouter au panier" directement). Attends que la page de détail du produit soit complètement chargée.

✅ Résultat attendu : La page fiche produit s'ouvre avec l'image agrandie, la description complète, le prix et un bouton d'achat.
Équivalent Playwright
await page.locator('.product-card').first().click();
await page.waitForLoadState('networkidle');
05
Assertion Vérification contenu fiche produit
Prompt Claude Computer Use

VÉRIFIE que la fiche produit contient bien les éléments suivants : — Nom du produit clairement visible (H1 ou titre principal) — Prix en € affiché — Image produit chargée (pas d'icône cassée) — Description ou texte de présentation — Un bouton "Ajouter au panier" (ou équivalent) Indique le nom du produit que tu vois et son prix.

✅ Résultat attendu : Tous les éléments présents. Claude retourne le nom et le prix exact du produit testé.
Équivalent Playwright
await expect(page.locator('h1')).toBeVisible();
await expect(page.locator('[class*="price"]')).toContainText('€');
await expect(page.locator('[class*="add-to-cart"]')).toBeEnabled();
06
Action Ajout au panier
Prompt Claude Computer Use

Clique sur le bouton "Ajouter au panier" (ou "Add to cart", "Acheter", "Commander"). Attends la confirmation visuelle de l'ajout (notification, animation panier, compteur mis à jour).

✅ Résultat attendu : Le produit est ajouté. Le compteur du panier (icône panier dans le header) passe à "1" ou une notification de confirmation apparaît.
Équivalent Playwright
await page.click('button[data-add-to-cart], .add-to-cart-btn');
await expect(page.locator('.cart-count, [class*="cart"] span')).toHaveText('1');
💡 Conseil formateur : C'est ici que la différence avec Playwright est flagrante — Claude identifie le bon bouton même si le sélecteur CSS change, car il le reconnaît visuellement par son libellé.
07
Assertion Vérification panier mis à jour
Prompt Claude Computer Use

VÉRIFIE dans le header de la page : — L'icône panier affiche un compteur ≥ 1 — OU une notification de type "Produit ajouté" est apparue — OU le mini-panier latéral s'est ouvert avec le produit listé Prends une capture d'écran du header / panier et décris l'état actuel.

✅ Résultat attendu : Confirmation visuelle que le panier contient au moins 1 article.

Scénario 3 — Consultation & Modification du panier

SC-03

Vérifie que le panier est accessible, modifiable et affiche le bon total.

08
Navigation Accès à la page panier
Prompt Claude Computer Use

Clique sur l'icône panier dans le header (ou sur "Mon panier", "Voir le panier"). Attends que la page panier soit complètement chargée et liste les articles.

✅ Résultat attendu : Page panier affichée avec le produit ajouté précédemment, son prix unitaire et un total calculé.
Équivalent Playwright
await page.click('[href*="cart"], [href*="panier"], .cart-icon');
await page.waitForSelector('.cart-items, .woocommerce-cart-form');
09
Assertion Vérification contenu panier
Prompt Claude Computer Use

VÉRIFIE que la page panier affiche : — Le nom du produit ajouté (identique à celui de la fiche produit) — La quantité (doit être 1) — Le prix unitaire en € — Le total de la commande en € — Un bouton ou lien "Passer la commande" ou "Checkout" Confirme les montants affichés.

✅ Résultat attendu : Tous les éléments présents. Total = quantité × prix unitaire (calcul cohérent).
10
Saisie Modification de la quantité
Prompt Claude Computer Use

Dans la ligne du produit dans le panier, trouve le champ quantité. Change la valeur de 1 à 2. Clique sur le bouton "Mettre à jour le panier" (ou "Update cart", ou appuie sur Entrée si le champ se met à jour automatiquement). Attends que le total se recalcule.

✅ Résultat attendu : Le total est maintenant égal à 2× le prix unitaire. Le compteur panier dans le header affiche "2".
Équivalent Playwright
await page.locator('input[name="cart[...][qty]"]').fill('2');
await page.click('[name="update_cart"]');
await page.waitForResponse(r => r.url().includes('cart'));

Scénario 4 — Tunnel de commande (Checkout)

SC-04

Simule le remplissage du formulaire de commande. ⚠️ S'arrête avant la confirmation de paiement réel.

11
Navigation Accès au checkout
Prompt Claude Computer Use

Depuis la page panier, clique sur le bouton "Passer la commande" (ou "Checkout", "Commander", "Valider mon panier"). Attends que la page de commande se charge (formulaire de facturation visible).

✅ Résultat attendu : Page checkout chargée avec formulaire d'adresse de facturation et section de paiement.
Équivalent Playwright
await page.click('.checkout-button, [href*="checkout"]');
await page.waitForSelector('#billing_first_name, form.checkout');
12
Saisie Remplissage adresse de facturation
Prompt Claude Computer Use

Remplis le formulaire de facturation avec les données de test suivantes : — Prénom : Jean — Nom : Testeur — Adresse : 12 Rue de la Qualité — Ville : Bruxelles — Code postal : 1000 — Pays : Belgique — Téléphone : +32 470 000 000 — Email : test.e2e@exemple.be Prends une capture d'écran du formulaire rempli avant de continuer.

✅ Résultat attendu : Tous les champs remplis sans message d'erreur visible. Les données sont bien saisies.
Équivalent Playwright
await page.fill('#billing_first_name', 'Jean');
await page.fill('#billing_last_name', 'Testeur');
await page.fill('#billing_email', 'test.e2e@exemple.be');
// ... etc
⚠️ Important : Utiliser uniquement des données de test fictives. Ne jamais saisir de vraies coordonnées personnelles ou bancaires dans un test E2E.
13
Assertion Vérification récapitulatif de commande
Prompt Claude Computer Use

VÉRIFIE la section récapitulatif de commande sur la page checkout : — Le produit correct est listé avec la bonne quantité — Le sous-total correspond à ce qui était dans le panier — Les frais de livraison sont affichés (ou "livraison gratuite" mentionnée) — Le total final TTC est visible Indique les montants exacts que tu vois.

✅ Résultat attendu : Récapitulatif cohérent avec le panier. Total TTC calculé correctement.
14
Assertion Vérification modes de paiement disponibles
Prompt Claude Computer Use

VÉRIFIE et liste tous les modes de paiement disponibles sur la page (ex : carte bancaire, virement, PayPal, Bancontact, Mollie...). VÉRIFIE qu'au moins 1 mode de paiement est sélectionnable. NE PAS cliquer sur "Confirmer la commande" — le test s'arrête ici pour éviter une transaction réelle.

✅ Résultat attendu : Au moins 1 option de paiement visible et sélectionnable. Le bouton "Passer la commande" est présent mais NON cliqué.
💡 Conseil formateur : Ce point illustre une limite des tests E2E sur environnement de production — on ne peut pas déclencher un vrai paiement. En projet réel, on utilise un environnement de staging avec des numéros de carte de test Stripe/Mollie.

Scénario 5 — Cas négatifs & Robustesse

SC-05

Vérifie le comportement de l'application face aux erreurs et cas limites.

15
Cas négatif Checkout sans données obligatoires
Prompt Claude Computer Use

Retourne à la page checkout (https://hyla-air-solution.be/checkout/). Sans remplir aucun champ du formulaire, clique directement sur le bouton "Passer la commande". VÉRIFIE que des messages d'erreur de validation apparaissent sur les champs obligatoires. Liste les champs en erreur et le message affiché.

✅ Résultat attendu : Messages d'erreur sur les champs obligatoires (prénom, nom, email, etc.). Commande NON passée.
Équivalent Playwright
await page.click('[name="woocommerce_checkout_place_order"]');
await expect(page.locator('.woocommerce-error, .invalid')).toBeVisible();
16
Cas négatif Format email invalide
Prompt Claude Computer Use

Dans le champ Email du formulaire checkout, saisis une valeur invalide : ceci-nest-pas-un-email Clique en dehors du champ ou essaie de soumettre le formulaire. VÉRIFIE qu'un message d'erreur apparaît indiquant que l'email est invalide.

✅ Résultat attendu : Message d'erreur de validation sur le champ email. La commande ne peut pas être finalisée.
17
Cas négatif Panier vide — accès direct au checkout
Prompt Claude Computer Use

Vide le panier (supprime tous les articles) ou accède directement à https://hyla-air-solution.be/checkout/ avec un panier vide. VÉRIFIE que le site : — Affiche un message "Votre panier est vide" (ou équivalent) — OU redirige vers la page panier/boutique — N'affiche PAS un formulaire de commande utilisable

✅ Résultat attendu : Le site gère correctement l'accès checkout avec panier vide et guide l'utilisateur vers les produits.
💡 Conseil formateur : Ce scénario teste la robustesse du tunnel de commande face aux manipulations directes d'URL — un cas souvent oublié dans les tests manuels.
18
Assertion Recherche produit inexistant
Prompt Claude Computer Use

Utilise la fonction de recherche du site (icône loupe dans le header). Saisis un terme qui ne devrait retourner aucun résultat : xyzproduitinexistant99 VÉRIFIE que le site affiche un message de type "Aucun résultat" ou "Aucun produit trouvé" et ne génère pas d'erreur technique (pas de page blanche, pas de code d'erreur HTTP 500).

✅ Résultat attendu : Message "aucun résultat" affiché proprement. L'interface reste stable et utilisable.
